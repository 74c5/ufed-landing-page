    Scrolling Modes

    Here we have a choice of 3 scrolling modes:

    • Auto: Uses 'window.scrollBy' function with scroll option '{behavior: "auto"}' to scroll after a nav click. This should result in a near instantaneous jump to the target section.
    • Smooth: Uses 'window.scrollBy' function with scroll option '{behavior: "smooth"}' to scroll from the nav. This would be the 'default' way to achieve scrolling in an app.
    • Custom: Utilises a custom built method to scroll. Initially, this was built because I hadn't figured out a way to disable the scroll event handler reliably during a menu commanded scroll. I kept it because it allows more fine grained control over the the scroll. Also, smooth scrolling seems to sometimes glitch for me. I've followed an ease-in-out approach; starting the scroll slowly for the first frame and then accelerating (bigger jumps per frame) towards the middle and then slowing down again. There is still some work to be done to make the underlying code production ready - but it was an interesting exercise.
    • scrollIntoView: Uses the experimental Html.scrollIntoView method. Not production code, but was the first method I played with. Not considered suitable, because it does not allow for adding an offset to the scroll amount - so the top of the element will be hidden under the nav bar. Interesting, because it is the simplest of the methods; it can be called on the target element directly.
